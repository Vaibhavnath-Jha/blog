NAS#

Create a mount point mkdir /mnt/nas-drive

add entry in fstab (nano /etc/fstab) to automatically add drive on boot /dev/sdX1 /mnt/nas-drive ext4 defaults 0 2

mount the drive mount -a

update packages and install sambda apt update && apt install samba -y

create the samba config file nano /etc/samba/smb.conf

add the following config, the drive will be called as NAS when attaching as network drive. yourusername takes the value of computer name. If not known, find out by whoami.

[NAS]
path = /mnt/nas-drive
read only = no
browsable = yes
guest ok = no
valid users = yourusername

Add the user and set a new password smbpasswd -a yourusername

restart the samba daemon systemctl restart smbd

The drive should be visible and can be attached using the following address. Use the IP addr of the host \\192.168.1.x\NAS

Email Server#

  1. Login to your gmail account
  2. Setup 2FA if not already done
  3. Go to the appPasswords page
  4. Create a new app password and save the key somewhere
  5. Use the following setting wherever you want to setup an email server
SMTP Server Address: smtp.gmail.com

Use Authentication: yes

Secure Connection: TLS/SSL based on your mail client/website SMTP plugin

SMTP Username: your Gmail account (xxxx@gmail.com)

SMTP Password: <The app password you created>

Gmail SMTP port: 465 (SSL) or 587 (TLS)
  1. Test the settings by sending an email
